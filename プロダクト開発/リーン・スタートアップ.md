# 目的
+ 今TaskLunnerというプログラミング独学者向けの学習進捗管理サービスを作っている
+ このサービスが成功するかどうか予想したい。また成功確率を上げるための方法があるなら知りたい
+ どうすれば、お金を支払ってでも使いたいサービスを作れるのか知りたい

# 問い
+ お金を払ってでも使用したいサービス、無料なら使いたいサービス、無料でも使用したいと思わないサービスを分ける要素は何か
+ なぜ売れるために一生懸命考え、開発したサービスなのに、全く売れないなんてことがあるのだろうか
+ サービスを世に出してから、どれくらいで見切りをつけるのか
+ そのサービスが売れるかどうかについて、アプリケーションの機能や品質がどれほど影響を与えるだろうか。
+ 努力していることのうち、価値を生み出していることとそうでないものは何か
+ 価値を生み出している努力とは何か。

# わかっていること
+ 多くのサービスは、大体3から6ヶ月程度の期限を決めてリリースする
+ どれだけ素晴らしい戦略を立てても、それが当たるかどうかは分からない
+ めちゃくちゃ売れるかもしれないし、「誰も欲しがらない製品を作ってしまう」かもしれない
+ 製品・機能の開発に費やした結果、それにどれだけの価値があるかはリリースしないと分からない
+ リリースしたら、その評価が降る
+ どれだけ改良を重ねても、売り上げが伸びない
+ 顧客は我々が想定したようには動かないかもしれない
+ 顧客はすでに代替のサービスを使っているかもしれない。新しいものに乗り越えようと思わないかもしれない。
+ 顧客は本当にその課題を解決したいと思っていないかもしれない
+ 一生懸命書いたコードも、時間も、コストも無駄になるかもしれない

# 学び

+ 市場に投下するまではあらゆることの語尾は、「〜かもしれない」だらけ
+ 市場に投下し、世の中からの評価が下り、「〜だった」に変わる
+ 世の中に投下して、すぐ答えがでるわけではないということ。
+ 早期の学びを得たからこそ、「最終的に」成功を収めた
+　　何が顧客にとって価値を見出すのか学ぶこと
+　　ビジネスとは限られたリソースの中で活動する必要がある。その中で成果を出す癖をつけないといけない。
+　　顧客が願っていることに合わせて、上手に仕事をした結果、売り上げがついてきた
+　作りたいものを如何にマネタイズするかではなく、つくるべきものを究明すること
+　数字からスケールするかどうか評価することができてしまう
+　　やることが増えれば増えるほど、それが無駄になる可能性も増える

## 僕の場合
### プログラミングを学ぶ人にとって、お金を払うだけの価値があることとは何か?
+ スクール(確実に転職する機会・環境にお金を払う) 
+ 本(教材がないと学ぶことができない)
+ 動画教材（本より手軽に学べる。学習コストの軽減にお金を払う）
+ MENTA（独学ではやりきれないため、必要なときに相談できる人をつけることにお金を払う）
+ コミュニティー（モチベーション維持や転職に向けて有益な情報の収集・レビューを受けたりする。）
+ 実務経験が積めるサービス

### プログラミングビジネスの特徴
+ 「転職する」強い欲求があるものの、独学では心理的なハードルやコストが高い
+ よって、ハードルやコストを下げる
+ あるいは欲求が得られる確率をより強固にできるという期待
+ 要は、「転職が圧倒的に有利になりうる経験が積めること」
+ に対してお金が支払われている

### プログラミングを学ぶ人が、無料で使用しているサービスは何か
+ Qiita,zenn,ブログ(転職活動で有利になる。学習の質向上)
+ Twitter(転職活動で有利になる。モチベーション維持)
+ github(転職活動で有利になる。モチベーション維持)
+ 学習進捗管理(モチベーション維持)
+ プログラミング転職あるいはプログラミングを学ぶ過程で、役に立つものは無料で使う。

### もし有料の学習進捗管理サービスがあったら、使用するだろうか
#### 予測
+ 既存で無料のサービスがあり、かつ評判も良いためお金を支払ってまで使う可能性は低い。
+　乗り換えるコストも高い。よほどの理由がない限りは無料でも使用しない。
+　仮に既存サービスがなかったとしても、学習の進捗をとるだけのサービスにお金を支払うほどのメリットを感じるかは微妙
+　単に「進捗が取れてモチベーション維持が期待できるサービス」では、お金を支払ってくれない

### プログラミングを独学でやり遂げる人の悩みは全て解決したのか？
+ そんなことはない。だとしたら、スクールの需要は減っても良いはず。

### プログラミングを独学で学ぶ人の悩みは何か?
+ 基本的に一人で学習しているため、モチベーション維持できるか心配
+ わからないことを気軽に聞ける相手がいない
+ 自分がとっている選択肢が正しいのか分からない不安
+ ポートフォリオを作っても、プロの目線からレビューを受けられない
+ 学習は自分でできても、転職活動が上手くできるのか心配
+ チーム開発の経験が積められない

### 独学で頑張っている人が、スクールに通わないでも転職支援サポートを受けられないか
+ お金を払って、
+ 書類選考率UP対策（ポートフォリオ/github/自己紹介/）
+ 面接通過率UP対策
+ 転職活動の戦略立案支援

### プログラミング未経験から自分でサービスを作るノウハウが学べる講座などないか

#### 僕は自分が使う進捗管理サービスを作りたいと思っており、それがビジネスにならないか考えているのではないか?
+ おっしゃる通り。
+ 自分が欲しているから、どうすればこのサービスでマネタイズできるか一生懸命考えている。
+ でもこの発想は、恐らく危険信号だと感じる。
+ 実際に提供する価値をユーザーが必要とするのか、その価値がユーザーにとってお金を支払うだけのものなのか調べてないから。
+ お金になりそうな理由づけを探そうとしている。確証バイアスだ。
+ きっと、役に立つだろうし、他の人も良いサービスだね。と言っていくれてている。でもこれは、お金を払うだけの悩みを解決するサービスではないと思う。
